# Specs

## Purpose

이 디렉터리는 방법론 차원의 generic spec standard를 담는다.
특정 제품의 실제 feature package는 이 저장소가 아니라 각 제품 저장소가 소유한다.

이 디렉터리는 아래를 고정한다.

- feature package의 최소 구조
- package 안에서 어떤 파일이 어떤 책임을 가지는지
- package authoring workflow가 어떤 산출물에 도달해야 하는지

## Contains

- `SPEC_PACKAGE_STANDARD.md`

## Does Not Contain

- 제품별 feature package
- 제품별 architecture terms
- 제품별 approval policy
- 제품별 route decision
- 제품별 framing 문서
