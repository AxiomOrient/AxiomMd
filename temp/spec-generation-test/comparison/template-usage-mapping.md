# Template usage map (AxiomMd -> skill workflow)

- `templates/input.packet.yaml`
  - Entry: `intake-normalizer`, `scope-router`, `feature-package-author`, `spec-writing-standard`
  - Used for: `feature-package` / `scoped-artifact` / `review` / `workflow-step` selection and input contract
- `templates/route.decision.yaml`
  - Entry: `scope-router`
  - Used for: routing to direct-package vs framing-first vs hold
- `templates/handoff.packet.yaml`
  - Entry: `feature-package-author`, `package-readiness-review`
  - Used for: stage handoff, status propagation, next-step orchestration
- `templates/feature/*.yaml` and markdown (`package.yaml`, `requirements.yaml`, `invariants.yaml`, `design.md`, `tasks.md`, `evals.yaml`, `risks.yaml`)
  - Entry: `feature-package-author`
  - Used for: authoring contract package output truth
- `templates/reconcile.result.yaml`
  - Entry: `reconcile-review`
  - Used for: execution evidence reconciliation decision
- `templates/execution.plan.yaml`
  - Entry: execution planning/runtime workflows
  - Used for: execution plan packet handoff in pipeline
- `templates/authoring.handoff.packet.yaml`
  - Entry: feature-package handoff continuity
  - Used for: run artifact in `feature-package-author`
- `templates/workflow` docs (`system-blueprint.md`, `product-charter.md`, `readiness-report.md`)
  - Entry: `charter-blueprint-author`, `scope-router`, `package-readiness-review`
  - Used for: framing and readiness reporting documents
