# ESCALATION_POLICY.md

## Purpose

This document defines when execution must stop and who must decide next.

> Do not guess across ownership boundaries.

## Escalation Triggers

1. spec and design conflict
2. design and eval intent conflict
3. implementation passes logic but fails a contradictory test
4. invariant risk appears
5. task scope must expand beyond approval
6. boundary change is required
7. requirement meaning is unclear
8. blocking eval intent appears wrong
9. risk posture changes
10. another contributor cannot resume from repository files
