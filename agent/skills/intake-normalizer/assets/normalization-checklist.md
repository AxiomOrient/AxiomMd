# Normalization Checklist

- [ ] `request_summary`가 한두 문장으로 정리됐다.
- [ ] `target_kind`가 명시됐다.
- [ ] `scope.in`과 `scope.out`가 분리됐다.
- [ ] `constraints`가 비어 있지 않다.
- [ ] `done_signals`가 비어 있지 않다.
- [ ] `source_context_refs` 또는 `evidence_refs`가 있다.
- [ ] 모르는 것은 `open_questions`로 남겼다.
- [ ] output contract가 아직 미정이면 그 이유가 packet에 드러난다.
