# Document Map

## Purpose

이 저장소에서 어떤 문서가 어떤 개발 방법론 질문에 답하는지 고정한다.

## Document Set

| 문서 | 답하는 질문 |
| --- | --- |
| `docs/01_METHOD_TAXONOMY.md` | AI 개발 방법론들을 어떻게 구분하는가 |
| `docs/02_CONSTITUTION.md` | 이 방법론의 불변 원칙은 무엇인가 |
| `docs/03_CLOSED_LOOP.md` | 표준 실행 루프는 어떻게 닫히는가 |
| `docs/04_QUALITY_SYSTEM.md` | 코드 품질, 실패, 하네스를 어떻게 운영하는가 |
| `docs/05_SKILL_SYSTEM.md` | skill을 어떻게 정의하고 workflow 자산으로 쓰는가 |
| `docs/06_REPOSITORY_GUIDELINES.md` | 이 저장소에 무엇을 넣고 무엇을 넣지 않는가 |
| `specs/SPEC_PACKAGE_STANDARD.md` | 실행 가능한 spec package는 어떤 구조를 가져야 하는가 |
| `templates/**` | 새 package를 어떻게 바로 시작하는가 |

## Boundary

- 이 저장소는 방법론을 다룬다.
- 특정 제품의 blueprint나 feature truth는 다루지 않는다.
